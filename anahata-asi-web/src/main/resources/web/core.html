Anahata Core

An Enterprise ready, Domain-Driven abstraction of the Context Window. Engineered for stability, modularity, portability, observability, security and traceability.

Mission Statement

Core provides the atomic units of context (Messages, Parts, Tools) and the context augmentation and metabolic management systems (CwGC) required to sustain live, long-running, multi-modal, cross platform, super intelligent agentic sessions without provider lock-in.

The AGI Container

The Agi class is the root orchestrator. It manages the session blueprint (AgiConfig), high-performance Kryo serialization for passivation/activation, and the lifecycle of all context components.

  • Metabolic Policy: Defined in AgiConfig, setting the default "Turns to Keep" (Max Depth) for different part types: Text (108), Thoughts (12), Tools (12), and Blobs (4).
  • Transactional State: The container automatically snapshots session state (history, tool attributes, resources) to disk upon tool execution or turn completion.
  • Session Scope: The sessionAttributes map in ToolManager provides turn-persistent storage for tools, enabling stateful automation across a conversation.

Messages & Parts

Anahata V2 treats context as a rich graph. A conversation is a sequence of AbstractMessage turns, each containing a collection of AbstractPart objects tracking their own Sequential ID and Metabolic State.

In-Band Metadata Injection

Self-awareness is achieved by interleaving invisible metadata headers before every component. This allows the model to see its own Part ID, Remaining Depth, and Pruning State (Pinned, Pruned, or Auto).

Garbage Collection (CwGC)

The Context Window Garbage Collector manages the session's "Metabolism". It transitions parts through three states: ALIVE (Full Content), GHOST (Metadata only), and RECYCLED (Purged from JVM).

  • PINNED:
    Vital awareness (source code, instructions) that is IMMUNE to garbage collection.
  • AUTO:
    Metabolic state. Soft-pruned into a "Ghost" when Remaining Depth reaches 0.
  • PRUNED:
    Explicitly recycled content. Can be Unpruned back to life until the message turn is GC'd.

The GC Hard Trigger

A message turn only becomes Garbage Collectable (permanently purged from JVM memory) once every individual part within that message has reached an effectively pruned state.

Context Providers

The sensors of the ASI. Providers contribute high-salience, just-in-time information to the RAG Message at the end of every turn.

Reactive Events

Built on PropertyChangeSupport, the Core API enables zero-polling UI updates. The StatusManager fires events for transitions between IDLE, THINKING, and EXECUTING_TOOLS.

Toolkits & Tools

The ToolManager automates capability discovery. Tools are simple Java methods in @AiToolkit annotated classes. Anahata handles the reflective schema generation and deferred execution lifecycle.
