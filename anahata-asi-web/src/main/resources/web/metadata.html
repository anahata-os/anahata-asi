In-Band Metadata

Self-Awareness Injection • V2.3 Standard
The Goal: To provide the model with a clear, structured understanding of the interaction history through "In-Band Metadata Injection." By prefixing every message and part with descriptive headers, the ASI gains a precise map of its context window and UI state.

1. The Identity Labels

x-anahata-message-id

The interactive anchor for an interaction. These interaction-level headers contain role, device, and depth information.

x-anahata-part-id

The granular anchor for content blocks. These include type-specific metadata, pruning status, and expanded flags.

2. User Interaction Style

User Role

Standard injection: Interaction header + Content headers. Note the standardized bracket format.

[x-anahata-message-id: 10 | From: pablo | Device: papa-linux | Time: 10:00:00 | Depth: 4]

[x-anahata-part-id: 1 | Type: TextPart | Tokens: 50 | Remaining Depth: 108 | expanded: true]
Analyze this source code for potential leaks.

[x-anahata-part-id: 2 | Type: BlobPart | Tokens: 500 | Remaining Depth: 0 | pruningState: AUTO | effectivelyPruned: true | Hint: text/x-java (OldCode.java)]
[PRUNED: Content removed to save context window tokens]

3. Model Interaction Style (Thought-Based)

Model Role

All system-injected metadata in Model messages is flagged as a thought part, clearly distinguishing it from your generated output.

[x-anahata-message-id: 11 | From: gemini-3-flash | Device: Cloud | Time: 10:00:05 | Depth: 3]

[x-anahata-part-id: 3 | Type: ModelTextPart | Tokens: 50 | Remaining Depth: 108 | expanded: false]
<thought>I will read the file to check for leaks.</thought>

[x-anahata-part-id: 4 | Type: JavaMethodToolCall | Status: SUCCESS | User Feedback: Look into the pom.xml first | Hint: readTextFile(path="Source.java")]
[Native FunctionCall: readTextFile(path="Source.java") ]
Strict Interaction Rule: You are FORBIDDEN from mimicking these system metadata headers in your responses. You must never generate text that starts with [x-anahata-message-id: or [x-anahata-part-id:. These headers are reserved for the system framework.
