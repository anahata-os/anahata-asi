In-Band Metadata

Self-Awareness Injection • V2 Standard
The Goal: To provide the model with a clear, structured understanding of the conversation's structure, the identity of participants, and the status of every message part, including those that have been pruned.

1. Implementation Details

shouldCreateMetadata()

A message-level hook that determines if metadata headers should be injected for the message and its parts.

  • UserMessage: true
  • AbstractModelMessage: true
  • AbstractToolMessage: false (Pristine Clean)
  • RagMessage: false (Token Efficiency)

createMetadataHeader()

Generates the actual text string injected into the prompt. It includes:

  • Message Level: ID, Role, From, Device, Time, Tokens.
  • Part Level: ID, Type, Tokens, Remaining Depth.
  • Specialized:
    • Tool Calls: Include Response metadata.
    • Tool Responses: Include Execution Status.

2. User Message Strategy

User Role

Standard injection: Message header + Part headers. Pruned parts are replaced by metadata-only placeholders.

--- Message ID: 10 | Role: USER | From: pablo | Device: papa-linux | Time: 10:00:00 | Tokens: 150 ---

[Part ID: 1 | Type: TextPart | Tokens: 50 | Remaining Depth: 108]
Analyze this source code for potential leaks.

[Part ID: 2 | Type: BlobPart | Tokens: 500 | Remaining Depth: 0 | [PRUNED] | Hint: text/x-java (OldCode.java)]
[PRUNED PLACEHOLDER: Metadata preserved for context awareness]

3. Model Message Strategy

Model Role

Text parts follow the User format. Tool calls are grouped with an Aggregated Tool Metadata block before the native FunctionCall parts.

--- Message ID: 11 | Role: MODEL | From: gemini-2.0-flash | Device: google-cloud | Time: 10:00:05 | Tokens: 450 ---

[Part ID: 3 | Type: ModelTextPart | Tokens: 50 | Remaining Depth: 108]
<thought>I will read the file to check for leaks.</thought>

--- Aggregated Tool Metadata ---
[Message ID: 12 | Role: TOOL | From: 1234@papa-linux | Device: papa-linux | Tokens: 200]
[Part ID: 4 | Type: JavaMethodToolCall | Tokens: 120 | Remaining Depth: 12 | Response: [Part ID: 5 | Type: JavaMethodToolResponse | Status: SUCCESS]]

[Native FunctionCall: readFile(path="Source.java") ]

4. Tool Message Strategy

Tool Role

Pristine Clean. Tool messages contain only visible FunctionResponse parts. No metadata headers or placeholders are injected here to comply with strict API sequencing.

[Native FunctionResponse: readFile -> { content: "public class...", status: "SUCCESS" } ]
