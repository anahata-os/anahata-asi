NetBeans ASI Studio

Deep IDE integration for the ultimate Java coding experience.

The NetBeans module transforms the IDE into a high-fidelity ASI Container, giving the model direct access to project structures, compilation errors, and native refactoring APIs.

The Annotator Strategy

Anahata NetBeans provides a non-intrusive, high-fidelity overlay system that injects ASI awareness directly into the IDE's workflow. This is implemented via the AnahataAnnotationProvider.

Project Badges

Real-time visual indicators in the Projects view showing session counts and health metrics for Anahata-enabled projects.

Line Annotations

The ASI can "paint" feedback, suggestions, or errors directly onto specific lines in the Java editor using native Annotation objects.

Maven Lifecycle Integration

The Maven toolkit provides the ASI with deep hooks into the NetBeans Maven Embedder. This allows the model to reason about dependencies and execute project goals programmatically.

  • Dependency Resolve: Fully recursive transitive dependency analysis.
  • Goal Execution: Synchronous execution of clean, install, or custom profiles.
  • Pom Management: Surgical updates to pom.xml via the addDependency super-tool.

The Java Code Model

Leveraging the org.netbeans.api.java.source APIs, the CodeModel toolkit provides the ASI with a high-performance index of your entire codebase.

One-Shot FQN Resolution

Optimized for token efficiency, tools like getMembersByFqn allow the model to skip discovery turns when the type name is already known.

Hierarchy Analysis

Recursive subtype/supertype searching to understand implementation patterns and inheritance trees.

Smart Refactoring Bridge

The Refactor toolkit exposes NetBeans' industry-standard refactoring engine (Rename, Move, Copy, Safe Delete) as agentic tools. This ensures that when the ASI modifies code, all references across the entire project graph remain consistent.

// Example of an agentic refactoring call
refactor.rename(filePath, "NewClassName"); 
// Updates the file, references in all projects, and the filesystem in one shot.

The Development Lifecycle

For contributors building new tools, NetBeans ASI Studio supports a high-velocity nbmreload workflow. This allows developers to re-deploy the Anahata plugin into a target IDE instance without restarting, enabling real-time testing of new capabilities.
